# Contributing

Thanks for considering a contribution. This project is a workflow skill rather than a code library, so useful contributions usually improve clarity, trust, and repeatability.

## Good Contributions

- Better prompts for specific paper types or disciplines.
- Open-access demo cases with source links and reproducible outputs.
- Stronger QA checklists for metadata, numeric claims, figures, tables, and Word layout.
- Examples of common bad outputs and how to repair them.
- Terminology examples for ecology, climate, biomedical, AI, or other research fields.

## Contribution Principles

- Preserve scientific accuracy over narrative smoothness.
- Do not include copyrighted PDFs, copyrighted figures, or generated article drafts that cannot be redistributed.
- Keep internal audit materials separate from publishable article text.
- Prefer concrete examples over abstract advice.
- Explain why a workflow rule matters when adding a new rule.

## Pull Request Checklist

Before opening a pull request, check:

- Markdown files render cleanly.
- Links use relative paths when pointing inside the repository.
- New examples avoid copyrighted source material unless they are clearly licensed for reuse.
- New prompts preserve the skill's focus on metadata, methods, numeric audit, figure explanation, and WeChat-ready output.
- The change does not weaken scientific caution or encourage unsupported claims.

## Issues

When reporting a problem, include:

- the paper type or discipline;
- what prompt you used;
- what output failed;
- whether the issue concerns metadata, methods, numbers, figures, language style, or Word layout.
